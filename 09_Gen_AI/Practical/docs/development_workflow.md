# Development Workflow

1. Create feature branch.
2. Read the relevant phase README and task list.
3. Implement minimal version first.
4. Add tests.
5. Document tradeoffs.
6. Refactor only after correctness.
7. Keep API contracts explicit.

## Branch Naming
- `phase-01/<feature>`
- `phase-02/<feature>`
- `phase-03/<feature>`
- `phase-04/<feature>`
- `phase-05/<feature>`
- `capstone/<feature>`
